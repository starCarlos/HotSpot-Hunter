## 概述

HotSpot Hunter 使用 AI 模型分析新闻的重要性，帮助用户快速识别关键新闻。系统支持批量分析和单条分析两种模式。

## 批量分析提示词

### 提示词模板

系统使用以下提示词模板进行批量分析：

```
请分析以下多条新闻的重要性，返回一个JSON对象，格式如下：
{
    "results": [
        {"title": "新闻标题1", "importance": "critical" | "high" | "medium" | "low"},
        {"title": "新闻标题2", "importance": "critical" | "high" | "medium" | "low"},
        ...
    ]
}

重要性评级标准：
- critical（关键）: 对国家、社会、经济有重大影响，涉及重大政策、突发事件、重大事故等
- high（重要）: 对行业、地区有重要影响，涉及重要政策变化、重大商业事件等
- medium（中等）: 有一定关注度，但影响范围有限
- low（一般）: 普通新闻，关注度较低

特别注意：
- 股市行情类新闻（如"创业板指跌超1%"、"沪指涨0.17%"、"深成指跌0.44%"等日常股市波动）应评为"low"级别
- 仅包含股市指数涨跌、个股涨跌、板块涨跌等日常行情信息的新闻，不具有重大意义，应评为"low"

需要分析的新闻列表：
1. 标题: [新闻标题1]
   平台: [平台名称1]
   排名: [排名1]
2. 标题: [新闻标题2]
   平台: [平台名称2]
   排名: [排名2]
...

请为每条新闻分析重要性，返回完整的JSON对象，不要遗漏任何一条新闻。只返回JSON，不要其他内容。
```

### 提示词位置

提示词模板位于 `config/ai_analysis_prompt.txt`，可以根据需要自定义修改。

## 单条分析提示词

如果批量分析失败，系统会自动回退到单条分析模式，使用以下提示词：

```
请分析以下新闻的重要性，只返回一个JSON对象，格式如下：
{
    "importance": "critical" | "high" | "medium" | "low"
}

重要性评级标准：
- critical（关键）: 对国家、社会、经济有重大影响，涉及重大政策、突发事件、重大事故等
- high（重要）: 对行业、地区有重要影响，涉及重要政策变化、重大商业事件等
- medium（中等）: 有一定关注度，但影响范围有限
- low（一般）: 普通新闻，关注度较低

特别注意：
- 股市行情类新闻（如"创业板指跌超1%"、"沪指涨0.17%"、"深成指跌0.44%"等日常股市波动）应评为"low"级别
- 仅包含股市指数涨跌、个股涨跌、板块涨跌等日常行情信息的新闻，不具有重大意义，应评为"low"

新闻信息：
- 标题: [新闻标题]
- 平台: [平台名称]
- 排名: [排名]

请只返回JSON，不要其他内容。
```

## 重要性评级标准

系统使用四级评级标准：

- **critical（关键）**: 对国家、社会、经济有重大影响，涉及重大政策、突发事件、重大事故等
- **high（重要）**: 对行业、地区有重要影响，涉及重要政策变化、重大商业事件等
- **medium（中等）**: 有一定关注度，但影响范围有限
- **low（一般）**: 普通新闻，关注度较低

## 使用建议

1. **批量大小**: 建议每批 20 条新闻，平衡效率和准确性
2. **标题匹配**: AI 返回的标题必须与输入完全匹配，用于结果映射
3. **容错处理**: 如果批量分析失败，系统会自动回退到单条分析模式
4. **API 配置**: 确保 AI API Key 配置正确，参考 [config/README.md](README.md)

## 自定义提示词

如果需要自定义提示词，可以修改以下文件：

- **批量分析提示词**: `config/ai_analysis_prompt.txt`
- **重要性分析提示词**: `config/importance_analysis_prompt.md`（本文档）

修改后重启服务即可生效。
